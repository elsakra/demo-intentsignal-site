01 / 13

The medium is the message.

Five principles that every subsequent decision in this document defers to. If a tradeoff needs resolving on the build, rank choices against this list.
Principles
  • P1
    Show the plumbing. Visitors should see the pipeline run. The narrator panel, the side drawer, the "we ran this on YOU 47 seconds ago" — these are not flourish, they are the product pitch. Never hide the machinery.
  • P2
    Editorial, not SaaS. Newsreader at display size, generous leading, one accent color, no gradients, no glass, no glow. If it looks like a template, we cut it.
  • P3
    Dense information is the aesthetic. Small caps labels, mono annotations, numbered lists, rule lines. We are GTM engineers; the page reads like a well-kept README.
  • P4
    Personalization never breaks. Fallback tiers are a design surface. A generic hero that waits > 2s looks broken; a pre-baked archetype variant looks intentional.
  • P5
    Motion is surgical. One letter-level rewrite moment, one module crossfade, one panel reveal. Everything else is still. Motion signals "something just happened" — if we animate everything, nothing animates.

Usage rules

Do

Use red only for: live-indicator dots, matched/personalized affordances, italic accent in display headlines, CTA on dark sections, the one "matched to you" tag in the logo strip. Budget: no more than 4 red elements visible at once above the fold.

Don't

No red body text. No red on red. No red gradients. No red buttons on cream background — cream+red is the italic-accent only. CTA on cream is ink-on-cream with optional red 1px underline on hover.

Pairing rules

R1 · Italic is accent

Newsreader italic (light, 300) is reserved for the emotional verb in a headline — leaking, rewrites itself, costing. Always one italic phrase per headline, never two.

R2 · Mono is evidence

Anything the user should read as "machine output, not marketing" is JetBrains Mono — narrator, drawer, arch nodes, metric proofs, footer flex line.

R3 · No mixing mid-line

Never mix serif and mono inside one sentence. Mono is its own line, its own block. Serif and sans may share a block (hero H1 + deck).

06 / 13

Nav, hero, dual logo strip.

The signature asset is the dual logo strip — OUR CLIENTS on top, RECENT OPPS CREATED on bottom, vertical divider between small-caps label and logos. It is the single most important visual on the page and must read correctly even before personalization fires.

Nav bar

Sticky from scroll-top. Height 64px. Cream background, no border at rest; on scroll past 8px, add 1px bottom border #D7CFBE and a 16px backdrop-blur. Logo mark (square with centered red dot, 24px) + wordmark intentsignal.ai in 13px JetBrains Mono, weight 500, left. Right cluster: Services Work Approach — each 13px Geist 500, 24px gaps, hover → 1px red underline 3px offset. Then a primary CTA pill Book a call ↗ in ink-fill-white-text, 36px tall. No mobile hamburger; collapse to horizontal scroll strip under 768px.

Hero layout

Full-width, content-column-constrained. 160px top padding, 120px bottom. H1 takes ~920px max-width, deck caps at 640px. CTA is a single button (48px tall, 20px horizontal padding, ink fill, white text, JetBrains Mono 11px caps) plus a secondary text link "or watch the 60-sec demo ↗" inline-right at 14px Geist. The narrator panel lives fixed bottom-right 24px from edges, 380px wide, 300-420px tall depending on expanded state. Does not cover hero content at ≥1280px; on smaller widths, it collapses to a 44px red pill ("▸ show what we just did to this page") that expands on click.

Claude stream typewriter — precise spec

Phase 1 (delete): existing characters of the generic H1 delete right-to-left, 40ms per char, with a 1px red caret trailing the deletion head. Caret fades when delete completes. Phase 2 (retype): Claude tokens arrive over HTTP stream; rendered char-by-char at 25ms, but if tokens arrive faster the renderer catches up by doubling-up chars until the queue is empty (never render faster than 12ms/char — it looks like a screen bug, not a rewrite). Subhead and CTA label run the same sequence with a 180ms and 360ms delay respectively (stepped reveal). If total elapsed exceeds 1.6s and stream is still incomplete, fall back to m-crossfade with the pre-baked archetype variant.

12 / 13

Fallback, demo mode, accessibility.

The personalization must never visibly fail in a sales-call screen-share. Three tiers of graceful degradation, one demo-override mode, and a documented accessibility baseline.

Tier 1 · Claude stream resolves < 1.5s

Full typewriter rewrite runs. Narrator shows all 7 lines with real latency. This is the A+ path and should fire for ~70% of visits.

Tier 2 · Identification resolves, stream > 2s

Skip typewriter. Crossfade in the closest pre-baked archetype variant (6 total: fintech·mid / fintech·large / saas·mid / saas·large / ecomm·mid / ecomm·large). Narrator S5 shows "✓ Loaded archetype: fintech·mid" instead of "Generating copy…".

Tier 3 · Identification fails entirely

Generic variant stays. Narrator fades in with "○ Visitor not identified. Showing default." in muted gray — an honest, on-brand failure that still feels intentional.

Demo-mode query param

?demo=company.com short-circuits the identification call and loads a hardcoded enrichment payload keyed to that domain. Fixtures live in /demo-fixtures/*.json. On the narrator, line S2 renders as "✓ Resolved: {fixture.name} at {fixture.company} (demo mode)" — the "(demo mode)" tag is red, so the sales rep and the prospect both see it's intentional. Add ?demo=company.com&quiet=1 to suppress the tag for screenshot-capture work.

Accessibility baseline

  • A1
    Reduced motion. prefers-reduced-motion disables m-typewrite entirely (content swaps via 200ms crossfade). Progress bar fills in one step, not animated.
  • A2
    Screen readers. Hero H1 is marked aria-live="polite", but with a 2s debounce so assistive tech doesn't read two hero variants in sequence during the typewrite.
  • A3
    Contrast. All body copy ≥ 7:1 against its background. Muted type (#6B6862 on cream) ≥ 4.6:1. Narrator dim text on #0E0E0E is #D6D1BF = 11.1:1.
  • A4
    Keyboard. Narrator panel is a <section aria-label>, reachable by tab; drawer trap-focuses and returns focus on close. Esc closes drawer. ? opens a keyboard-shortcut help card.
  • A5
    Visitor opt-out. Narrator's overflow menu includes "Don't identify me" — clears the pixel, reverts to generic, writes is_optout=true cookie. GDPR-friendly, and it's on-brand to show the visitor they're in control.
